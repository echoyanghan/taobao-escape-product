# 逃宝产品原型

逃宝是一款用于探索“消费冲动代偿”体验的独立产品原型。它让用户把购物车先搬进逃宝，完成一次不会真实扣款的模拟购买，第二天再判断自己是否仍然需要这些商品。

在线体验：<https://echoyanghan.github.io/taobao-escape-product/>

## 使用方式

1. 在首页上传购物车截图，或选择手动输入商品。
2. 将商品加入逃宝购物车并完成模拟结算。
3. 第二天进入“待确认”，选择“确认省下”或“认真买下”。
4. 在“我的逃宝”中查看最近决定与累计已省金额。

## 原型能力边界

- 本项目是前端交互原型，不是购物平台，也不提供商品销售、代购、支付、物流或金融服务。
- 截图导入通过阿里云百炼 Qwen OCR 提取图片中的文字、价格和位置，不连接淘宝账号或读取淘宝平台数据。
- OCR 结果可能存在误差，所有商品在导入前都需要用户核对；支付、优惠、订单、物流和金额统计仍为演示效果。
- 数据仅保存在当前浏览器的 `localStorage` 中。清理浏览器数据、更换设备或浏览器后，记录可能丢失。
- 原型展示不构成消费、健康、医疗、金融或其他专业建议。

## 隐私说明

使用智能识别时，浏览器会先压缩截图并删除原文件中的附加信息，再将压缩图片发送至逃宝的 Vercel 服务端，由服务端转发至阿里云百炼 Qwen OCR。API Key 仅保存在服务端环境变量中。逃宝不提供截图存储功能，识别完成后不会主动保存原图；第三方服务对请求数据的处理仍受其服务条款与隐私规则约束。

请不要选择含有姓名、手机号、收货地址、订单编号、头像或其他个人信息的截图。若未来接入云存储、数据分析或账号系统，应在上线前另行提供完整的个人信息处理规则，并重新评估告知、同意、最小必要、保存期限和安全措施。

## Vercel 部署与 OCR 配置

1. 在 Vercel 中导入本 GitHub 仓库，Framework Preset 选择 `Other`，其余构建设置保持默认。
2. 在 Project Settings → Environment Variables 中填写 `DASHSCOPE_API_KEY` 与 `DASHSCOPE_WORKSPACE_ID`，并应用到 Production、Preview 和 Development。
3. 可选填写 `QWEN_OCR_MODEL=qwen3.5-ocr`。未填写时服务端会使用该默认模型。
4. 重新部署后，通过 Vercel 域名体验截图识别。GitHub Pages 只能托管静态页面，不能运行本项目的 `/api/recognize-cart` 服务端函数。

请勿把真实 API Key 写入代码、`.env.example`、Issue、截图或提交记录。服务端对上传图片做类型与体积校验，并关闭 API 响应缓存；正式开放给更多用户前仍应补充频率限制、用量告警和隐私政策。
